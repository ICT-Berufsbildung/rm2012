Regionalmeisterschaft IIIII0III00

This document describes the tasks for the Regionalmeisterschaft 2012. You have 175 minutes to complete as many tasks as possible. Each task has only a short description, you’re expected to apply your knowledge and experience to implement your own solutions.

The project is a web shop for the region near Eiger, Mönch & Jungfrau. Customers can browse on the website through the categories to different products. The products can be added to a cart and ordered in the checkout.

Parts of the website are not finished yet and need to be implemented by you. You are free to use and extend existing parts of the project or implement your own.

Rating Criteria

Each task gets rated by the implementation of the required functionality. In addition the following criteria apply to all tasks and must match your solution:

All code needs to be nicely formatted, indented and commented where needed. PHP functions have to be documented inline (except getter and setter). The generated output is needed to comply to the HTML5 specification. SQL queries have to be protected against injection. No errors must be issued with the PHP error reporting level set to E_NOTICE. Database access, input handling and output generation have to be separated. The web shop must practice good usability principles.

Task I: Random Product

The product displayed on the current home page is statically hardcoded in the template (views/home.php).

Your task, should you decide to accept it, is to display a new random product after each page reload on the home page. All product information needs to be fetched from the database. The product image and title link to the detail page of the product.

Task I0: Order Tables

Please study the database tables carefully. There are no tables to store orders from the customer

Your task, should you decide to accept it, is to create two new tables orders and order_items to store orders in the database and add them to the file create.sql.

An order consists of a unique identification and a creation date. In addition they contain the name, the address, the phone number and the email address of the customer.

Each order can have multiple order items. And order item is always linked to one product and a request quantity.

Keep in mind that product prices and names need to be changed in the future without having any side effects on the existing orders. Add the missing field(s) to the appropriate table(s).

Task II: Checkout

The current checkout displays the form but does not have any functionality to save the order.

Your task should you decide to accept it, is to implement the PHP code in the file create.php to save the orders to the database tables created in Task I0. Before creating the order all values have to be validated. All fields are required, the email address has to contain @ and the phone numbers must match this regular expression:

^([0-9\(\)\/\+ \-]*)$

In case of a validation error inform the customer and display the checkout again.

Task I00: Ajax Cart

Removing a product from the cart reloads the page and therefor destroys all quantity modifications of other products.

Your task should you decide to accept it, is to write JavaScript code to execute this action with Ajax. After a successful response the product has to be removed from the cart table. All quantity modifications of the other products don’t get lost during this operation.

Task I0I: Wish List (Bonus)

This bonus task has no existing functionality and includes different skills (HTML, JavaScript and PHP).

Your task, should you decide to accept it, is to implement a wish list for products. The customer can add a product on its detail page to the wish list. After a click on a button an Ajax call is made to save the product in the session. A link in the site header leads to a separate page where all products in the wish list are displayed. The customer can also remove a product from the wish list.
